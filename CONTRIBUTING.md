## How to Contribute

### Reporting Bugs üêõ

If you find a bug, please create an issue with:
- A clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Example SQL query that causes the issue
- Browser version and OS

### Suggesting Features üí°

We welcome feature suggestions! Please create an issue with:
- Clear description of the feature
- Use case / why it would be useful
- Examples of how it would work

### Code Contributions üîß

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow existing code style
   - Add comments for complex logic
   - Test thoroughly
4. **Commit your changes**
   ```bash
   git commit -m "Add: description of your changes"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Create a Pull Request**
   - Describe what changes you made
   - Reference any related issues

## Development Setup

1. Clone the repository
2. Open Chrome and go to `chrome://extensions/`
3. Enable "Developer mode"
4. Click "Load unpacked" and select the extension folder
5. Make changes to the code
6. Click the refresh icon on the extension card to reload

## Testing

Before submitting a PR, please test:
- Format various SQL queries (SELECT, INSERT, UPDATE, DELETE, etc.)
- Test with different formatting options
- Test minify functionality
- Test copy to clipboard (both plain and HTML)
- Test in different scenarios (long queries, complex CASE statements, etc.)

## Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Keep functions focused and small
- Follow existing code patterns
